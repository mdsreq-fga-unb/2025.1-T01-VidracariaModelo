    1. Lista de requisitos

    Requisitos Funcionais (RF)

    ID TÍTULO DESCRIÇÃO REGRA DE NEGÓCIO OBJETIVO ESPECÍFICO
    RF01 Selecionar data e horário para agendamento O sistema deve permitir que o cliente selecione data e horário disponíveis para realizar agendamentos. RN02 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF02 Notificar agendamento O sistema deve fornecer notificações sempre que um novo agendamento for feito para garantir acompanhamento. RN04 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF03 Cadastrar agendamento O sistema deve permitir que os administradores cadastrem agendamentos para registrar compromissos com clientes. RN05 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF04 Listar agendamentos do dia O sistema deve listar os agendamentos realizados para os administradores visualizarem os compromissos do dia. RN01 Objetivo 3: Viabilizar gestão de demandas.
    RF05 Remarcar agendamento O sistema deve permitir a remarcação de agendamentos para reprogramar compromissos com clientes. RN04 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF06 Cancelar agendamento O sistema deve permitir o cancelamento dos agendamentos para ajustes na agenda de compromissos. RN09, RN19, RN03 Objetivo 3: Viabilizar gestão de demandas.
    RF07 Notificar sobre alterações em agendamentos O sistema deve enviar notificação caso um agendamento seja alterado ou cancelado para informar o cliente e o administrador. RN06 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF08 Criar orçamento O sistema deve permitir a criação de orçamentos para registrar propostas de serviços no site. RN07 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF09 Listar orçamentos O sistema deve listar orçamentos solicitados para facilitar o acompanhamento das propostas. RN09 Objetivo 3: Viabilizar gestão de demandas.
    RF10 Editar orçamentos O sistema deve permitir edição de valores, prazos e observações para atualizar os dados do orçamento conforme necessário. RN10, RN08 Objetivo 2: Otimizar gestão de orçamentos e agendamentos.
    RF11 Notificar expiração de orçamento O sistema deve enviar alertas ao cliente e à equipe quando um orçamento estiver próximo de expirar para garantir o acompanhamento. RN08 Objetivo 3: Viabilizar gestão de demandas.
    RF12 Listar vendas O sistema deve apresentar a lista de vendas feitas pela empresa para controle comercial. RN11 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF13 Visualizar venda O sistema deve permitir a visualização dos detalhes da venda para consulta de informações. RN10 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF14 Editar venda O sistema deve permitir edição dos detalhes da venda para correção de dados. RN11 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF15 Registrar recibo O sistema deve permitir o registro de recibos com dados do serviço prestado para gerar comprovantes. RN10 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF16 Editar recibo O sistema deve permitir alteração dos dados registrados em um recibo para correções. RN12, RN13 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF17 Exportar recibo O sistema deve gerar arquivo PDF com os dados do recibo preenchido para emissão de comprovantes. RN10, RN14, RN12 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF18 Registrar despesas O sistema deve permitir que usuários registrem novas despesas para controle financeiro. RN13 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF19 Categorizar despesas O sistema deve classificar despesas por tipo para facilitar a organização financeira. RN13 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF20 Editar despesas O sistema deve permitir edição de despesas conforme as permissões para correção ou atualização de registros. RN15 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF21 Exportar despesas O sistema deve gerar relatório em PDF com os dados registrados das despesas para controle e análise financeira. RN16 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF22 Registrar interações com cliente O sistema deve registrar todas as interações feitas com o cliente para acompanhamento do histórico de atendimento. RN16 Objetivo 3: Viabilizar gestão de demandas.
    RF23 Contatar empresa O sistema deve permitir comunicação por WhatsApp, formulário e redes sociais para facilitar o contato com os clientes. RN02 Objetivo 1: Sofisticar o Marketing proporcionando posicionamento online.
    RF24 Permitir gerenciamento da FAQ O sistema deve permitir que os administradores gerenciem as perguntas frequentes para auxiliar o cliente. RN17 Objetivo 1: Sofisticar o Marketing proporcionando posicionamento online.
    RF25 Excluir Avaliação O sistema deve permitir a exclusão de avaliações para manter a integridade das informações publicadas. RN17 Objetivo 1: Sofisticar o Marketing proporcionando posicionamento online.
    RF26 Publicar avaliação O sistema deve permitir que o usuário publique uma avaliação com nota e comentário para registrar sua opinião sobre os serviços. RN17, RN18 Objetivo 1: Sofisticar o Marketing proporcionando posicionamento online.
    RF27 Criar perfil do cliente O sistema deve permitir a criação de perfis de cliente para armazenar seus dados e histórico de interações. RN20, RN15 Objetivo 3: Viabilizar gestão de demandas.
    RF28 Editar informações do cliente O sistema deve permitir a edição das informações básicas de identificação e contato dos clientes para manter os dados atualizados. RN20 Objetivo 3: Viabilizar gestão de demandas.
    RF29 Excluir informações do cliente O sistema deve permitir a exclusão das informações do perfil do cliente para atender solicitações de privacidade. RN20 Objetivo 3: Viabilizar gestão de demandas.
    RF30 Gerar gráfico de desempenho O sistema deve gerar um gráfico com métricas como orçamento, conversões e atendimentos para apoiar o planejamento estratégico. RN17 Objetivo 4: Aprimorar gestão financeira e de vendas.
    RF31 Gerar gráficos comparativos financeiros O sistema deve gerar gráficos de receitas vs despesas para apoiar a tomada de decisão financeira. RN17, RN14 Objetivo 4: Aprimorar gestão financeira e de vendas.

    Requisitos Não Funcionais (RNF)

    ID Nome do Requisito Descrição do Requisito Classificação URPS
    RNF1 Consultar histórico de despesas As consultas ao histórico de despesas devem ter tempo de resposta inferior a 2 segundos. Performance
    RNF2 Garantir Lei Geral de Proteção de Dados O sistema deve garantir a segurança das informações e a conformidade com a Lei Geral de Proteção de Dados (LGPD). Implementação
    RNF3 Adaptar a diferentes telas A interface deve se adaptar dinamicamente aos tamanhos de tela: 720x1280, 1080x1920 e 1920x1080. Interface
    RNF4 Exibir categorias A interface da página principal deverá introduzir os usuários às principais informações para as demais navegações: Portfólio da vidraçaria, além do rodapé e cabeçalho com as demais navegações. Usabilidade
    RNF5 Exibir calendário com disponibilidade de agendamento O sistema deve exibir calendário com dias e horários já ocupados e disponíveis para agendamento. Usabilidade
    RNF6 Exibir resumo do orçamento O sistema deve exibir um resumo do orçamento solicitado pelo cliente. Usabilidade
    RNF7 Exibir FAQ O sistema deve exibir uma área com dúvidas frequentes sobre serviços e processos da empresa. Usabilidade
    RNF8 Exibir produtos e serviços cadastrados no portfólio online da empresa O sistema deve exibir os produtos e serviços disponíveis no portfólio online da empresa. Usabilidade

    Regras de Negócio (RN)

    ID Regra de Negócio
    RN01 Os agendamentos listados tem que ter sido feitos no máximo com uma semana.
    RN02 Agendamentos só podem ser feitos em horários disponíveis e dentro do expediente da empresa.
    RN03 Clientes podem cancelar agendamentos com no mínimo 12h de antecedência.
    RN04 Agendamentos cancelados geram notificações para o cliente e o administrador.
    RN05 Apenas administradores podem criar ou consultar agendamentos de outros usuários.
    RN06 Somente administradores podem editar agendamentos enviados ao cliente.
    RN07 O sistema deve alertar automaticamente sobre agendamentos não confirmados nos próximos 3 dias.
    RN08 Os orçamentos devem expirar em no máximo 15 dias.
    RN09 Orçamentos devem estar associados a um cliente e podem ser filtrados por cliente, status, etc.
    RN10 Vendas e recibos devem estar relacionados a um cliente e orçamento.
    RN11 Apenas administradores podem criar, editar ou excluir recibos já registrados.
    RN12 Vendas e recibos podem ser exportados em PDF com os dados registrados.
    RN13 Apenas administradores podem registrar, editar ou excluir pagamentos.
    RN14 Toda despesa registrada deve ser categorizada para compor relatórios financeiros.
    RN15 Todo serviço (venda e atendimento) deve ser associado ao cliente.
    RN16 Somente administradores podem incluir ou editar anotações internas no perfil do cliente.
    RN17 Somente administradores podem criar e/ou consultar gráficos de desempenho.
    RN18 Clientes podem avaliar serviços após sua conclusão.
    RN19 Os agendamentos podem ser cancelados por administradores e pelos clientes.
    RN20 Apenas os administradores podem criar, editar ou excluir perfis de cliente.
